# === NOTES AND COMMENTS ===
# 
# TTL (Time To Live) Guidelines:
# - 60-300 seconds: For records that change frequently (testing, load balancing)
# - 3600 seconds (1 hour): Standard for most records
# - 86400 seconds (24 hours): For stable records like NS records
# - 604800 seconds (7 days): For very stable infrastructure records
#
# Priority Guidelines for MX records:
# - Lower numbers = higher priority
# - Common values: 10, 20, 30 for primary, secondary, tertiary
# - Google Workspace typically uses: 1, 5, 5, 10, 10
#
# SPF Record Mechanisms:
# - a: Allow sending servers listed in A records
# - mx: Allow sending servers listed in MX records  
# - ip4/ip6: Allow specific IP addresses
# - include: Include SPF record from another domain
# - ~all: Soft fail for unlisted servers
# - -all: Hard fail for unlisted servers
#
# DMARC Policies:
# - none: Take no action (monitoring only)
# - quarantine: Mark suspicious emails
# - reject: Reject emails that fail authentication
#
# SRV Record Format: priority weight port target
# - Priority: Lower number = higher priority
# - Weight: Load distribution among same priority
# - Port: Service port number
# - Target: Hostname providing the service
